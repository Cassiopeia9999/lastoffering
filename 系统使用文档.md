# 校园失物招领智能管理系统操作说明

## 1. 文档说明

本文档面向系统普通用户与管理员，重点说明系统的访问方式、页面入口、主要操作步骤及常见问题处理方法。

说明如下：

- 本文档以当前代码实现与当前页面交互为准。
- 若工作日志、早期设计稿或需求文档与现版本存在差异，应以当前系统实际功能为准。
- 文档写法偏向“操作手册”，适合演示、答辩和后续交付时直接使用。

## 2. 系统访问方式

### 2.1 运行后访问地址

开发环境常用访问地址如下：

- 前端页面：`http://localhost:5173`
- 后端接口文档：`http://localhost:8000/docs`
- 后端健康检查：`http://localhost:8000/health`

若采用 Docker Compose 部署，通常通过以下地址访问：

- 前端入口：`http://localhost`
- 后端接口前缀：`http://localhost/api/v1`

### 2.2 系统角色

系统主要包含两类角色：

- 普通用户：可注册登录、发布失物或招领信息、以图搜物、智能寻物、留言、发起匹配、确认认领、查看通知、维护个人资料。
- 管理员：除普通用户能力外，还可进入后台进行用户管理、物品管理和统计查看。

## 3. 普通用户操作说明

### 3.1 注册账号

操作步骤如下：

1. 打开系统首页，进入“登录”页面。
2. 点击“立即注册”进入注册页面。
3. 填写用户名、密码、确认密码和联系方式。
4. 点击“注册账号”完成注册。

使用说明：

- 联系方式建议填写手机号、微信号或邮箱。
- 联系方式不会在公开列表中直接展示。
- 只有在双方确认匹配后，系统才会向相关用户推送联系方式通知。

### 3.2 登录系统

操作步骤如下：

1. 进入登录页面。
2. 输入用户名和密码。
3. 点击“登录”。
4. 登录成功后进入系统主页。

登录成功后，系统会自动保存当前登录状态，并在后续请求中附带身份令牌。

### 3.3 首页与导航使用

登录后可通过顶部导航快速进入以下页面：

- `以图搜物`
- `发布信息`
- `智能寻物`
- `快速发布`
- `我的发布`
- `个人中心`
- `我的通知`

首页还提供快捷入口卡片，可直接跳转到常用功能页面。

### 3.4 发布信息

“发布信息”适用于手动填写完整表单的场景，适合信息较明确、希望自行控制内容的用户。

操作步骤如下：

1. 点击顶部导航“发布信息”。
2. 选择发布类型：失物或招领。
3. 上传物品图片。
4. 按需使用图片裁剪功能调整展示区域。
5. 填写标题、描述、地点、时间、类别、颜色、品牌、关键词等信息。
6. 如需要，可点击“重新识别”获取新的图片分类建议。
7. 确认内容无误后点击“发布信息”。

系统处理说明：

- 图片上传后，系统会自动进行类别识别与特征提取。
- 若图片特征提取成功，后台会自动执行一次智能匹配。
- 若发现高相关候选结果，系统会通过通知提醒用户。

### 3.5 快速发布

“快速发布”适用于用户不想手动逐项填写表单，希望直接用一句话描述物品信息的场景。

操作步骤如下：

1. 点击顶部导航“快速发布”。
2. 在输入框中直接输入自然语言描述。
3. 可选用语音转文字后再交由系统解析。
4. 提交后，系统会自动提取类别、颜色、品牌、地点、关键词、标题和描述等字段。
5. 用户检查系统生成的结构化表单内容。
6. 必要时手动补充或修改信息。
7. 完成后提交发布。

示例描述：

- “我在图书馆丢了一个黑色小米充电宝”
- “在一食堂门口捡到一把带蓝色挂绳的钥匙”

使用建议：

- 句子中尽量包含地点、颜色、品牌、显著特征等线索。
- 若自动解析不完整，可手动补全后再发布。

### 3.6 以图搜物

“以图搜物”适用于用户已有物品照片，希望通过图片检索相似物品的场景。

操作步骤如下：

1. 点击顶部导航“以图搜物”。
2. 上传待检索图片。
3. 选择检索方向。
4. 按需调整返回数量和相似度阈值。
5. 提交后查看系统返回的相似物品列表。

结果说明：

- 系统会先识别图片类别，再提取特征向量。
- 检索结果会显示相似度分值及候选物品信息。
- 若结果较少，可尝试降低阈值重新检索。
- 若仍未找到目标物品，建议直接转入发布流程，等待后续匹配通知。

### 3.7 智能寻物

“智能寻物”适用于用户希望通过自然语言直接表达丢失线索，由系统自动完成筛选和排序的场景。

操作步骤如下：

1. 点击顶部导航“智能寻物”。
2. 输入一段自然语言描述。
3. 可选使用语音转文字。
4. 提交后查看系统解析出的检索条件与候选结果。

系统会重点识别以下信息：

- 物品类型
- 类别
- 品牌
- 颜色
- 地点
- 关键词和外观特征

结果页面通常会展示：

- 系统提取后的筛选条件
- 系统返回的说明性回复
- 按相关度排序的物品列表
- 每条候选结果的匹配原因

### 3.8 查看物品详情

当用户在列表页、搜索结果页或通知页点击某条物品时，可进入物品详情页。

详情页可查看以下信息：

- 图片与标题
- 失物或招领类型
- 类别、颜色、品牌、地点、时间
- 详细描述
- 发布者信息
- 相似物品推荐
- 匹配状态与认领进度
- 留言记录

### 3.9 留言沟通

留言功能用于在物品详情页进行补充沟通。

操作步骤如下：

1. 进入某条物品详情页。
2. 在留言区域输入内容。
3. 点击提交留言。

补充说明：

- 留言适合询问特征细节、补充拾取地点或核实线索。
- 其他用户的留言会触发通知提醒。
- 系统支持留言查看与删除。

### 3.10 发起“疑似遗失”匹配申请

当失主浏览到某条招领信息，认为该物品可能属于自己时，可发起“疑似遗失”申请。

操作步骤如下：

1. 进入目标招领物品详情页。
2. 点击“疑似遗失”按钮。
3. 在弹窗中选择自己名下对应的失物记录。
4. 提交申请。

提交后效果：

- 系统创建匹配记录。
- 双方会收到相应通知。
- 匹配记录进入待确认状态。

### 3.11 确认匹配

当一方收到匹配申请后，可在详情页或关联流程中进行确认。

操作步骤如下：

1. 查看系统推送的匹配通知或进入物品详情页。
2. 核对物品图片、描述、地点、时间和显著特征。
3. 点击“确认匹配”。

确认后的系统行为：

- 匹配状态变为已确认。
- 系统向双方推送联系方式通知。
- 双方可进行线下联系和交接。

### 3.12 完成认领

线下交接完成后，双方需分别在系统中确认完成认领。

操作步骤如下：

1. 进入相关物品详情页或认领流程页面。
2. 点击“确认完成认领”。
3. 在弹窗中再次确认。

完成逻辑说明：

- 单方确认后，系统会记录该方已完成确认。
- 双方都确认后，匹配记录状态变为 `completed`。
- 对应物品状态自动变更为已完成，避免再次被错误匹配。

### 3.13 我的发布

“我的发布”页面用于集中管理本人发布的失物和招领信息。

可执行操作包括：

- 查看本人全部发布记录
- 按状态查看待认领、已匹配、已完成内容
- 编辑物品信息
- 下架物品
- 恢复已下架物品
- 手动关闭物品
- 删除不再需要的记录

常见使用场景：

- 信息填写有误时，进入编辑页面修改。
- 物品已找回或已处理完毕时，手动关闭。
- 临时不希望公开展示时，先下架，后续可恢复。

### 3.14 个人中心

“个人中心”用于维护账号资料。

可维护内容包括：

- 头像
- 昵称
- 真实姓名
- 学院
- 班级
- 个性签名
- 邮箱
- 联系方式

操作步骤如下：

1. 点击顶部导航“个人中心”。
2. 修改个人资料或上传新头像。
3. 保存更新内容。

建议保持联系方式真实有效，以便匹配成功后顺利联系。

### 3.15 通知中心

“我的通知”页面用于统一查看系统消息。

通知内容包括：

- 系统发现疑似匹配
- 匹配申请提醒
- 联系方式共享通知
- 留言提醒
- 认领完成提醒

常见操作：

- 查看通知详情
- 标记单条通知为已读
- 全部标记为已读

建议用户在发布信息后及时关注通知中心，以免错过匹配结果。

## 4. 管理员操作说明

### 4.1 登录后台

管理员使用管理员账号登录后，可进入后台管理页面。

后台主要入口通常包括：

- 数据概览
- 用户管理
- 物品管理

### 4.2 数据概览

“数据概览”页面用于查看平台整体运行情况，常见统计项包括：

- 注册用户数量
- 物品发布总量
- 匹配记录数量
- 成功认领数量
- 未读通知数量
- 近 30 天发布趋势
- 各类别物品数量统计

该页面适合用于答辩演示和系统运行情况说明。

### 4.3 用户管理

管理员可在“用户管理”页面执行以下操作：

- 查看用户列表
- 按用户名等条件搜索用户
- 查看用户注册时间与当前状态
- 启用或禁用用户
- 设为管理员或取消管理员身份
- 重置用户密码
- 删除用户

使用注意：

- 删除用户或禁用用户前，应先确认是否影响其未完成的认领流程。

### 4.4 物品管理

管理员可在“物品管理”页面执行以下操作：

- 查看全部物品信息
- 按类型、状态、上下架情况筛选
- 搜索标题或关键词
- 强制下架违规内容
- 恢复已下架内容
- 修改物品状态

适用场景：

- 清理无效、重复或违规信息
- 处理不合规图片或描述
- 保持平台数据整洁与可用性

## 5. 常见问题

### 5.1 注册成功后无法登录

可能原因：

- 用户名或密码输入错误
- 后端服务未正常启动
- 数据库连接异常

建议处理：

- 先确认输入内容是否正确。
- 再检查后端服务和数据库是否正常运行。

### 5.2 图片上传后没有检索结果

可能原因：

- 图片清晰度不足
- 当前库中暂无高相似度候选物品
- 相似度阈值设置过高

建议处理：

- 更换更清晰的图片重新检索。
- 适当降低阈值。
- 直接发布信息，等待系统后续自动匹配。

### 5.3 快速发布或智能寻物解析不完整

可能原因：

- 输入语句过短
- 缺少地点、颜色、品牌等关键信息
- 自然语言描述不够明确

建议处理：

- 尽量补充完整线索。
- 使用系统解析结果后手动修正表单。

### 5.4 没有收到认领相关通知

可能原因：

- 尚未形成有效匹配
- 通知未及时查看
- 当前账号不是相关物品发布者

建议处理：

- 进入“我的通知”查看是否有未读通知。
- 检查是否使用了正确账号登录。

### 5.5 为什么联系方式不会直接公开显示

这是系统的隐私保护设计。为避免用户联系方式被公开抓取或滥用，系统仅在双方确认匹配后才向相关用户推送联系方式。

## 6. 使用建议

- 发布信息时尽量上传清晰、完整的物品图片。
- 描述中应包含地点、时间、颜色、品牌和显著特征。
- 发布后及时关注“我的通知”。
- 完成线下交接后，记得双方分别点击“确认完成认领”，以形成完整闭环。

## 7. 结语

本系统围绕校园失物招领场景设计，既支持传统的信息发布与查询，也引入了以图搜物、自然语言智能寻物、快速发布和自动匹配等能力。按照本文档进行操作，用户可完成从发布、检索、沟通到认领闭环的完整流程。
