# 失物招领系统（FastAPI）—— 环境安装说明

---

## 一、创建 conda 环境

```bash
conda create -n lostfound python=3.10.13
conda activate lostfound
```

> 若终端无法识别 `conda`，可在 **Anaconda Prompt** 中执行，或运行：`conda init powershell` 后重启终端。

---

## 二、安装依赖

### 方式 A：快速启动（推荐）

仅安装核心依赖，系统可正常运行，AI 使用占位模式：

```bash
conda activate lostfound
cd F:\Last_offering\items
pip install -r requirements-minimal.txt
pip check
```

### 方式 B：分步安装（按指定版本）

```bash
conda activate lostfound

# 基础科学计算库
pip install numpy==1.24.4
pip install pandas==2.1.4
pip install scipy==1.11.4
pip install scikit-learn==1.3.2
pip install matplotlib==3.8.2

# FastAPI 核心及工具
pip install fastapi==0.124.4
pip install uvicorn[standard]==0.33.0
pip install pydantic==2.10.6
pip install pydantic-settings==2.1.0
pip install python-dotenv==1.0.1
pip install PyYAML==6.0.1
pip install python-multipart==0.0.20
pip install typing_extensions==4.6.0

# 数据库相关（MySQL + ORM + 迁移）
pip install sqlalchemy==2.0.25
pip install pymysql==1.1.0
pip install alembic==1.13.1

# 图像处理库
pip install pillow==10.2.0
pip install opencv-python==4.9.0.80

# 通用工具库
pip install requests==2.31.0
pip install tqdm==4.66.2

# JWT
pip install python-jose[cryptography]

# 密码加密（bcrypt 必须为 4.0.1，5.x 与 passlib 不兼容）
pip install passlib[bcrypt]==1.7.4
pip install bcrypt==4.0.1

# 依赖检查
pip check
```

### 方式 C：完整安装（含 TensorFlow / PyTorch）

需要 ai_module 或真实模型时，先完成方式 A 或 B，再按顺序安装：

```bash
# TensorFlow（需先固定 numpy）
pip install numpy==1.24.3
pip install tensorflow==2.13.0

# PyTorch GPU 版（CUDA 11.8）
pip install torch==2.2.1 torchvision==0.17.1 torchaudio==2.2.1 --index-url https://download.pytorch.org/whl/cu118

# 若无 GPU，使用 CPU 版
# pip install torch==2.2.1 torchvision==0.17.1 torchaudio==2.2.1

pip check
```

## 三、数据库配置

1. 启动 MySQL 服务
2. 创建数据库：
   ```sql
   CREATE DATABASE IF NOT EXISTS lostfound CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```
3. 将 `.env.example` 复制为 `.env`，填写 `DB_PASSWORD` 等配置

---

## 四、启动项目

### 终端 1：后端

```bash
conda activate lostfound
cd F:\Last_offering\items
python -m uvicorn backend.app.main:app --reload --host 0.0.0.0 --port 8000
```

**conda 无法识别时**，直接使用 Python 完整路径：

```powershell
cd F:\Last_offering\items
& "C:\Users\Kvyain\.conda\envs\lostfound\python.exe" -m uvicorn backend.app.main:app --reload --host 0.0.0.0 --port 8000
```

> 路径请根据本机 conda 安装位置调整（如 Anaconda 可能在 `C:\ProgramData\anaconda3\envs\lostfound\python.exe`）

### 终端 2：前端

```bash
cd F:\Last_offering\items\frontend
npm install
npm run dev
```

## 五、注意事项

1. **bcrypt**：必须使用 `bcrypt==4.0.1`，5.x 与 passlib 不兼容，会导致登录 500 错误。
2. **MySQL**：启动前确认 MySQL 服务已运行，`lostfound` 数据库已创建。
3. **ai_module**：未安装 TensorFlow 时，`/api/v1/ai/*` 接口不可用，其余功能正常。
4. **占位模式**：`services/ai_classifier.py` 与 `ai_feature.py` 默认使用占位实现，接入真实模型后将 `USING_PLACEHOLDER = False` 即可。

## 六、项目结构概览

```
items/
├── backend/              # 后端（FastAPI）
│   └── app/
│       ├── api/v1/       # 接口路由
│       ├── core/         # 配置、数据库、安全
│       ├── crud/         # 数据操作
│       ├── models/       # ORM 模型
│       ├── schemas/      # 请求/响应模型
│       └── services/     # AI 服务（分类、特征、检索）
├── frontend/             # 前端（Vue3 + Vite + Element Plus）
├── ai_module/            # 独立 AI 模块（TensorFlow，可选）
├── .env                  # 环境变量（需自行配置）
├── requirements.txt
├── requirements-minimal.txt
└── 环境安装书.md         # 本文件
```
